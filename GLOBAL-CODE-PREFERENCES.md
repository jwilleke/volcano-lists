# Global Preferences

In all interactions and commit messages:

- **READ** AGENTS.md file and update as required
- **Be concise** and sacrifice grammar for concision
- **DRY** (Don't Repeat Yourself) principle in Code and Documentation
  - In Documentation, refer and link to other Documents
- **Iterate Progressively** — start with core features only, gather feedback
- **Present** a list of unresolved questions to answer, if any
- **Questions, Comments and Suggestions** are always encouraged
- **GitHub CLI** is the primary method for interacting with GitHub
- **On larger objectives** present phased implementation plan
- **NEVER** put unencrypted "Secrets" in Git or other CMS systems
- **Always** update docs/project_log.md as a log of work done on the project
- **Think first** and provide options
- In Markdown documents:
  - **bold** in list beginnings
  - Do not use "1. Headings" — ok to use "## Step 1 ..."
