## Project Structure

```text
volcano-lists/
├── data/                    # Volcano data files
│   ├── *.xls                # GVP source spreadsheets (committed)
│   └── volcanoes.json       # Generated by import (gitignored)
├── src/
│   ├── types.ts             # Volcano, SearchFilters, SearchResult interfaces
│   ├── import.ts            # XLS → JSON importer (npm run import)
│   ├── parser.ts            # Loads volcanoes.json at runtime
│   ├── search.ts            # Full-text search + faceted filtering
│   └── index.ts             # Interactive CLI entry point
├── docs/
│   └── project_log.md       # Session history and next steps
├── .claude/                 # Claude MCP commands
├── .github/workflows/       # CI/CD pipelines
├── dist/                    # Compiled output (gitignored)
└── AGENTS.md                # Project context (init file for AI agents)
```

## Directory Conventions

- **data/** — Source XLS files from GVP and generated JSON. XLS files are committed for reproducibility; `volcanoes.json` is gitignored since it's generated
- **src/** — All TypeScript source code. Flat structure (no subdirectories needed at this scale)
- **docs/** — Project log and supplementary documentation
- **dist/** — TypeScript compilation output

## Technology Stack

- **Runtime:** Node.js (v18+)
- **Language:** TypeScript (strict mode)
- **Package Manager:** npm
- **Spreadsheet parsing:** [xlsx](https://www.npmjs.com/package/xlsx) (SheetJS)
- **CLI interface:** Node.js built-in `readline`

## Key Dependencies

- **xlsx** — Parses GVP .xls files (XML SpreadsheetML format) during import

## Data Pipeline

```text
GVP Website (.xls download)
    ↓
data/*.xls (committed to repo)
    ↓
npm run import (src/import.ts)
    ↓
data/volcanoes.json (gitignored, ~2,661 records)
    ↓
npm run dev (src/index.ts loads JSON, provides interactive search)
```

The importer auto-detects epoch (Holocene/Pleistocene) from the filename and handles the differing column order between the two file types.

## Common Patterns

### Data Flow

All data is read-only. XLS files are parsed once during import. The app loads the resulting JSON into memory at startup and filters in-place.

### Search Architecture

Search uses simple in-memory filtering — no index, no database. The dataset is small (~2,700 records) so linear scan with string matching is fast enough. `search.ts` exports pure functions that take the full dataset and filters, returning matched results.
