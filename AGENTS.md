## Context Overview

- Project Name: `volcano-lists`
- Description: A searchable TypeScript application for exploring Global Volcanism Program (GVP) volcano data. Reads Holocene and Pleistocene volcano spreadsheets and provides search/filter capabilities across volcano name, country, region, type, rock type, tectonic setting, and more.

## Key Decisions

- Data source: GVP .xls files (XML SpreadsheetML format) from <https://volcano.si.edu/>
- Data pipeline: XLS → `npm run import` → `data/volcanoes.json` (gitignored) → app reads JSON
- Stack: TypeScript, Node.js, xlsx library for spreadsheet parsing
- Search: Client-side full-text and faceted filtering across all volcano fields
- No database — dataset is small (~2,700 records), read-only, fits in memory as JSON

## Commands

- `npm run import` — Parse XLS files from `data/` into `data/volcanoes.json`
- `npm run dev` — Run the interactive CLI search app
- `npm run build` — Compile TypeScript to `dist/`
- `npm run start` — Run compiled app
- `npm run typecheck` — Type-check without emitting
- `npm run lint` — Run code + markdown linting

## Data Update Workflow

- Download new Holocene/Pleistocene XLS files from <https://volcano.si.edu/>
- Place them in `data/` (filenames must contain "Holocene" or "Pleistocene")
- Run `npm run import` to regenerate `data/volcanoes.json`
- The importer auto-detects epoch from filename and handles the differing column order

## Volcano Data Schema

Each volcano record has 14 fields + epoch tag:
volcanoNumber, volcanoName, country, volcanicRegionGroup, volcanicRegion,
volcanoLandform, primaryVolcanoType, activityEvidence, lastKnownEruption,
latitude, longitude, elevation, dominantRockType, tectonicSetting, epoch

Note: Holocene and Pleistocene XLS files have Tectonic Setting and Dominant Rock Type columns swapped — the importer handles this.
